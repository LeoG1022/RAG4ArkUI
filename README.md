# RAG4ArkUI

> 面向 OpenHarmony / ArkUI-X 的本地化 RAG 代码生成与迁移系统。

## 项目愿景

`RAG4ArkUI` 是一套**纯本地**的检索引擎，专门服务 ArkUI-X / OpenHarmony 开发场景。完整规约见 [`docs/RAG4ArkUI-完整技术方案.md`](docs/RAG4ArkUI-完整技术方案.md)。三大差异化护城河：

- **领域语料**：官方文档 + 代码示例 + 迁移规则 + XDB 错误回流
- **本地化部署**：单 Rust 二进制 + ONNX Runtime + LanceDB / Tantivy，零云依赖
- **协议统一**：MCP（Claude Code / Cursor）+ HTTP（IDE 插件）+ LSP，同一二进制四种形态

# Agent Harness Template (本仓库基础设施)

本仓库同时是一个 **AI-agent 协作基础设施模板**——所有写操作（包括上面 RAG4ArkUI 产品代码）必须通过 agent harness 的归档与校验流程。完整规约：

- 用户视角 → 继续阅读本文件
- AI agent 入口 → [`AGENTS.md`](AGENTS.md)
- Claude Code SOP → [`CLAUDE.md`](CLAUDE.md)
- 新 agent 入门 → [`ONBOARDING.md`](ONBOARDING.md)

A reusable AI-agent collaboration infrastructure template. Clone it, run one command, and your project gains a full AI-agent operating framework with:

- **Skill system** — structured workflows that AI agents follow step-by-step
- **Archiving system** — every AI-driven change leaves a traceable record
- **Mechanical validation** — pre-commit hooks enforce consistency automatically
- **Multi-tool compatibility** — works with Claude Code, Cursor, Aider, Cline, Windsurf, Continue, opencode, and more

## Quick Start (5 steps)

```bash
# 1. Clone into your project root (or copy files into an existing project)
git clone <this-repo> my-project && cd my-project

# 2. Initialize hooks and verify structure
bash scripts/init-project.sh

# 3. Add your project-specific content
#    - Edit scripts/check-api-parity.sh  → add your code rules
#    - Edit scripts/classify-change.sh   → add your business directory paths
#    - Edit CLAUDE.md                    → update IDE table, directory overview

# 4. Add your first skill (optional — example skill included)
#    cp .claude/skills/example-skill.md .claude/skills/my-skill.md
#    # Edit it, then:
#    bash scripts/regenerate-skill-table.sh

# 5. First commit (hooks are now active)
git add . && bash scripts/commit.sh -m "Initial: add agent harness"
```

## Directory Structure

```
YourProject/
├── .claude/
│   ├── skills/           ← Skill definitions (trigger AI workflows)
│   │   ├── SKILL_SCHEMA.md   ← Frontmatter spec
│   │   └── example-skill.md  ← Template for new skills
│   └── references/       ← API mappings, best-practice guides (loaded on demand)
│       └── example-mapping.md ← Template for new mapping files
├── scripts/              ← Framework scripts (validation, archiving, hooks)
├── feedback/
│   ├── meta/             ← Meta-change archives (rules/interfaces/process, numbered)
│   └── features/         ← Business feature evolution archives
├── stats/                ← Agent commit token & change statistics
└── reports/              ← Output reports (auto-generated by skills)
```

## Key Concepts

### Skills
A **Skill** is a Markdown file in `.claude/skills/` that defines a structured AI workflow. When a user types `/skill-name`, the AI follows the steps exactly. Skills have:
- YAML frontmatter (name, description, version, feature_log flag)
- Step-by-step instructions the agent must follow
- Mandatory preflight (`bash scripts/preflight.sh`) as first step
- Mandatory `classify-change.sh` call in summary step

### Archiving
Two archive types live under `feedback/`:
- **`feedback/meta/`** — numbered records of changes to rules, scripts, skills, or any framework file. Required whenever an AI agent modifies a "meta" file. Generated via `bash scripts/new-feedback.sh <slug>`.
- **`feedback/features/`** — per-feature evolution logs. Required whenever an AI agent modifies business code. Generated via `bash scripts/new-feature.sh <name>` or `bash scripts/new-feature-log.sh <name> <slug>`.

Archives are **immutable** — the pre-commit hook blocks deletion or renaming.

### Pre-commit Hook
Installed by `install-hooks.sh`, it enforces:
1. `check-api-parity.sh` — your project-specific code rules (FAIL blocks commit)
2. `classify-change.sh` — meta changes must have a feedback file; business changes must have a feature log
3. `check-consistency.sh` — 19 structural consistency rules (FAIL blocks commit)

## Customization Checklist

After cloning, adapt these files to your project:

| File | What to change |
|---|---|
| `scripts/check-api-parity.sh` | Add project-specific code rules (P-*/R-*/S-*/C-* IDs) |
| `scripts/classify-change.sh` | Add your business directory paths to the `business` case |
| `CLAUDE.md` | Update directory overview, IDE entry table, Skill table (auto-synced) |
| `AGENTS.md` | Update subdirectory index to match your actual project structure |
| `feedback/DESIGN.md` | Document your architectural decisions |
| `.claude/skills/` | Delete example skill, add your own workflows |
| `.claude/references/` | Delete example mapping, add your project's reference tables |

## Adding a New Skill

```bash
# 1. Copy the example template
cp .claude/skills/example-skill.md .claude/skills/my-workflow.md

# 2. Edit the frontmatter and step content
# Required fields: name, description, version, feature_log

# 3. Sync the skill table in CLAUDE.md
bash scripts/regenerate-skill-table.sh

# 4. Verify consistency
bash scripts/check-consistency.sh
```

## Token Statistics

Every AI agent commit is tracked:

```bash
# After each agent commit (automatic via post-commit hook):
bash scripts/log-tokens.sh --from-commit HEAD

# Annotate with real token counts from your UI:
bash scripts/log-tokens.sh annotate <round> --input-tokens N --output-tokens N

# View reports:
bash scripts/stats-report.sh --by-round
bash scripts/stats-report.sh --trend
```
