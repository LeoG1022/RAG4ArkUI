## Plan

> 必备节（FAIL 级 M-FEATURE-PLAN）

### 设计：3 层（依赖反转）

```
            ┌─ Embedder, Retriever, Reranker, QueryEnhancer ─── core trait
            ├─ HybridRetriever (retrieval)
            ├─ InMemory/Lance/Tantivy (storage)
            ↑   trait object 注入
            │
   AppState ─→ server::http
            ↑
            │   CLI 构造好后传入
            │
   cmd_serve_http (cli) —— 复用 build_vector_load / build_bm25 / build_reranker / build_embedder / build_enhancer
```

server crate 不依赖 cli；cli 依赖 server。AppState 持 trait object，server 不直接绑后端类型。

### 路由设计（4 个端点）

| 路径 | 方法 | 真活/Stub | 用途 |
|---|---|---|---|
| `/health` | GET | ✅ | 健康检查 + 配置摘要（embedder/dim/bm25/vector/rerank/hyde） |
| `/corpus/list` | GET | ✅ | 列 corpus/ 子目录 + 文档数 |
| `/search` | POST | ✅ | 端到端检索（支持 enhance_query / expand_parent） |
| `/index` | POST | ⏳ stub | 触发索引（Day 14 仅返回提示，Week 4 续接异步任务） |

### Request / Response Schema

```rust
struct SearchRequest {
    query: String,
    k: Option<usize>,        // 默认 5
    enhance_query: bool,     // 默认 false
    expand_parent: bool,     // 默认 false
}

struct SearchResponse {
    hits: Vec<HitDto>,       // chunk_id + score + source + citation + content_preview + parent_*
    latency_ms: u128,
    embedder: String,
    bm25: String,
    rerank: Option<String>,
    enhancer: String,
}
```

`HitDto` 含：
- chunk_id / score / source (vector/bm25/hybrid/reranked)
- citation (source / heading_path / line_range / score)
- content_preview (前 500 字符)
- parent_preview / parent_chunk_id（启用 expand_parent 时）

### 错误处理

`HttpError(RagError)` 实现 `IntoResponse`，按 RagError 变体映射 HTTP status：
- `NotImplemented` → 501
- `Config` → 400
- 其他 → 500

返回 JSON `{ "error": "...", "kind": "..." }`。

### CLI 复用

CLI `Serve` subcommand 接受与 `Query` / `Eval` 相同的参数集：
- `--embedder mock|onnx` / `--model-path`
- `--bm25 memory|tantivy`
- `--vector memory|lancedb`
- `--rerank none|mock|onnx` / `--reranker-model-path`
- `--hyde none|mock`
- `--pre-rerank-k`
- 加 `--addr 127.0.0.1:7654`

server 不重新实现流水线构造，CLI 调 `build_embedder` / `build_bm25` / `build_reranker` / `build_enhancer` / `build_vector_load` 等已有函数 → 一致行为。

### 优雅退出

axum::serve 加 `with_graceful_shutdown(shutdown_signal())`：
- 监听 SIGTERM (`unix`) / SIGINT (Ctrl-C)
- 一收信号关闭 listener，已建连接处理完成才退出
