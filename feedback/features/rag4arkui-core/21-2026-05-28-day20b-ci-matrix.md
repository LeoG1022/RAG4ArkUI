## 本轮目标

把 Day 20a 的本地 host 打包脚本推进 CI，实现：
- tag `v*` 触发 → GitHub Actions 4 平台 matrix build

### 4 平台 matrix 设计

| target | runner | 备注 |
|---|---|---|
| `aarch64-apple-darwin` | `macos-14` | Apple Silicon · 与本地 dev 一致 |
| `x86_64-apple-darwin` | `macos-13` | Intel Mac · macos-13 已被标 deprecate · 后续切跨编 |
| `x86_64-unknown-linux-gnu` | `ubuntu-latest` | 主流服务器 |
| `x86_64-pc-windows-msvc` | `windows-latest` | shell: bash 显式（git-bash） |

**关键决策**：不跨编 · 每个 runner 上原生 build。`rustc -vV host:` 就是目标 triple。

### 跨平台细节

- **打包格式统一 tar.gz**（windows 也用 · Win10+ 内置 tar.exe 能解 · 简化 yml）
- **`BIN_SUFFIX=".exe"`** in `release-local.sh` for windows · cp + INSTALL 时正确处理
- **`shell: bash`** 显式声明 · windows runner 默认 PowerShell · git-bash 路径已内置
- **`fail-fast: false`** · 一个平台失败不阻塞其他（第一次 release 尤其有用）

### Workflow 拓扑

```
on: push.tags=v*  +  workflow_dispatch
    ↓
build matrix (4 jobs · parallel)
    each:  checkout → rust toolchain → rust-cache → release-local.sh → upload-artifact
    ↓
release job
    download-artifact (merge-multiple)
    → compute SHA256SUMS (consolidated across 4 platforms)
    → softprops/action-gh-release@v2 (create Release + upload 4 tar.gz + SHA256SUMS)
```

### 第一次 release 已知 risk

| risk | 缓解 |
|---|---|
| Windows tantivy build 路径未实测 | `fail-fast: false` · 不阻塞其他 3 平台 |
| `softprops/action-gh-release` `fail_on_unmatched_files: true` 卡 release | 4 平台 artifact 命名固定 · 与 release-local.sh 输出对齐 |
| Tag 写错（如 `0.0.2` 不带 `v`） | workflow 不触发 · 用户能立即发现 |
| 第一次 cargo build cold 10+ min | rust-cache 第二次跑后大幅加速 |

### 替代方案权衡（被否）

- 备选 1：用 `cargo-dist` 自动化（业界标准 · uv / starship 用）
  - 否决：workspace 要加 metadata + 远程拉工具 · 本轮范围内不必 · 手撸 yml + 复用 release-local.sh 更显式
- 备选 2：macOS 全用 macos-14 + 跨编 x86_64
  - 否决：macos-13 当前还能用 · 跨编 macOS 需要 SDK 配置 · 留 follow-up
- 备选 3：把 onnx 也进默认 matrix
  - 否决：~300MB ONNX Runtime 跨平台分发复杂 · Day 20c 独立处理
- 备选 4：用 actions/setup-rust 而不是 dtolnay/rust-toolchain
  - 否决：dtolnay/rust-toolchain 是 Rust 社区事实标准 · 更轻量
