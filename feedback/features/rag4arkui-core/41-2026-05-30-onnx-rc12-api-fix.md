### 错误分类（47 错 4 类）

| 类别 | 数量 | 修法 |
|---|---|---|

| `Session::run` 改 `&mut self` | ~2 | 包 `Mutex<Session>` · encode/score 内 lock |

### 关键 helper：generic ort_err

```rust
/// rc.12 起 ort 用 typed `Error<T>`（如 `Error<SessionBuilder>`）· 含
/// `NonNull<*>` / `dyn Any` 等非 Send/Sync 字段 · `?` 自动 From 转 anyhow
/// （要 Send + Sync）失败 · 必须显式 map_err。
fn ort_err<E: std::fmt::Display>(prefix: &'static str)
    -> impl FnOnce(E) -> anyhow::Error
{
    move |e| anyhow::anyhow!("{}: {}", prefix, e)
}
```

Generic E + Display bound 兼容 ort 所有 Error 类型变体（`ort::Error` / `Error<SessionBuilder>` / 等）。

### 关键改动：Session 包 Mutex

rc.12 起 `Session::run(&mut self, ...)` · 不再 `&self`。但 `Embedder::encode(&self, ...)` 是 trait 签名 · 不能改。

```rust
// Before
pub struct EmbeddingModel { session: Session, ... }

// After
pub struct EmbeddingModel { session: std::sync::Mutex<Session>, ... }

// encode 内：
let mut session = self.session.lock().map_err(...)?;
let outputs = session.run(ort::inputs![...]).map_err(ort_err("..."))?;
```

`std::sync::Mutex` 是 Send + Sync · 适合 `Arc<EmbeddingModel>` 跨线程 `spawn_blocking`。RerankerModel 同样改。

### 替代方案权衡

| 方案 | 优点 | 缺点 | 选 |
|---|---|---|---|
| A · helper + Mutex（本轮）| 最小改动 · 兼容现有 Embedder trait | helper 略丑 · Mutex 在 batch encode 锁竞争 | ✅ |
| B · 改 trait 为 `&mut self` | 干净 | 涉及 core / retrieval / cli 多 crate · 改动巨大 | ❌ |
| C · `RefCell<Session>` 内部可变 | 没 Mutex 开销 | 不 Send · spawn_blocking 不能用 · 死路 | ❌ |
| D · 新建 Session per call | 无锁 | 加载 ONNX 模型重 · 每次 100ms+ · 性能死 | ❌ |

选 A · 实测 5.65 秒编完 · `Mutex` 锁竞争只在 encode 调用边界 · 实际 encode 内推理已经是 CPU bound · 锁开销可忽略。
