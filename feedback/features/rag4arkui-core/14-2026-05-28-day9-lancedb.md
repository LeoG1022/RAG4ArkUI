### 设计：LanceVectorStore + VectorBackend 抽象

**Arrow Schema**（4 字段）：
| 字段 | 类型 | 用途 |
|---|---|---|
| `id` | Utf8 | 精确查找 + upsert delete-by-id |
| `content` | Utf8 | 完整 chunk 内容（stored） |
| `metadata_json` | Utf8 | 完整 ChunkMetadata JSON 序列化 |
| `vector` | FixedSizeList&lt;Float32, dim&gt; | 向量主体 |

与 TantivyBM25Index 的 `meta_json` 设计一致 —— 单字段 JSON 序列化避免逐字段 deserialize。

**关键算法**：
- `upsert`：先 `table.delete("id IN (...)")` 再 `table.add(batch)` 实现 upsert 语义
- `search`：`table.query().nearest_to(vec).limit(k*2).execute()` → 反序列化 + post-filter
- LanceDB 返回的 `_distance` 列（升序，越小越相似）→ 转 score = 1 - clamp(distance, 0, 1)
- `delete`：`table.delete("id IN ('a','b')")`（SQL-style predicate）
- `len`：`table.count_rows(None)`

**Day 9 简化**：
- Filter 在 LanceDB 内部仅做基本 SQL（id IN）；platform / tags / api_version / deprecated 等复杂多值过滤在反序列化后 **post-filter**（与 InMemoryVectorStore 一致行为）
- 复杂下沉到 Arrow expression 留 Day 9 续

### CLI 集成：VectorBackend abstraction

```rust
enum VectorBackend {
    Memory(Arc<InMemoryVectorStore>),
    #[cfg(feature = "lancedb")]
    Lancedb(Arc<arkui_rag_storage::LanceVectorStore>),
}

impl VectorBackend {
    fn as_store(&self) -> Arc<dyn VectorStore> { ... }
    fn embedder_model_id(&self) -> &str { ... }
    fn dim(&self) -> usize { ... }
    fn name(&self) -> &'static str { ... }
    async fn persist(&self, json_path: &Path) -> Result<()> {
        match self {
            Memory(s) => s.save_to(json_path).await?,   // JSON 落盘
            Lancedb(_) => {}                              // 自动 persist
        }
    }
}
```

CLI 三 subcommand 都加 `--vector memory|lancedb`，默认 memory 向后兼容 Day 8 之前。

## 改动要点

> API 选型 / 算法 / 关键决策

**与 Day 10 的差异**：
- crate 数 9（不变）
- 测试数 默认 44（不变）+ lancedb feature 加 5 → **49 with lancedb**
- 全 feature 启用约 64（含 onnx +1 / tantivy +7 / typescript +8 / lancedb +5 / dispatcher +4 等）

**API 选型**：
- `LanceVectorStore::open(uri, model_id, dim)` —— open or create 语义
- `embedder_model_id()` / `dim()` 静态访问（用于 CLI persist 调用）
- 实现 `VectorStore` + `MetadataStore` 双 trait（与 InMemoryVectorStore 对称）

**关键决策**：
- VectorBackend 抽象既保持 trait object 灵活（as_store 返回 `Arc<dyn VectorStore>`），又保留具体方法访问（embedder_model_id / persist）
- Arrow `FixedSizeListArray::from_iter_primitive` 简化向量批量构造
- delete-before-add 实现 upsert（LanceDB 不内置 upsert）
- post-filter 与 InMemoryVectorStore 完全一致行为（passes_post_filter 函数复制）
