## Plan

> 必备节（FAIL 级 M-FEATURE-PLAN）

### 6 job 设计

| Job | 类型 | 关键决策 |
|---|---|---|
| `check` | push + PR | `cargo check --workspace --all-targets`；快速基线 |
| `test` | push + PR | `cargo test --workspace --no-fail-fast`；跑 24 个测试 |
| `fmt` | push + PR | `cargo fmt --all --check`；零配置门禁 |
| `clippy` | push + PR | `cargo clippy --workspace --all-targets -- -D warnings`；严格 lint |
| `smoke` | push + PR + `needs: check` | `bash scripts/demo-smoke.sh --verbose`；端到端 CLI 行为 |
| `onnx-build` | **仅 workflow_dispatch** + 复选框 input | `cargo check --features onnx`；ort 拉库慢，手动触发 |

### 工程化要点

1. **并发取消**：`concurrency.cancel-in-progress: true` 同分支 push 取消旧 CI
2. **缓存策略**：`Swatinem/rust-cache@v2` 缓存 cargo registry + target；首跑 ~10 分钟，后续 ~3 分钟
3. **timeout 上限**：每 job 5-30 分钟，防止失控烧 CI 额度
4. **RUSTFLAGS 局部化**：不在 env 全局设 `-D warnings`（会让第三方依赖 warning 也 fail）；仅 clippy job `-- -D warnings`
5. **onnx job 复选框**：`workflow_dispatch.inputs.run_onnx_build` 让用户在 GitHub UI 上勾选触发，避免无意义运行

### 与现有自动化的边界

| 验证手段 | 触发 | 覆盖 |
|---|---|---|
| pre-commit hook（已有） | 每次 commit | meta 归档 + check-consistency 19 条 |
| `make smoke`（Day 2.5 已有） | 开发者手动 | 端到端 CLI 行为（local） |
| **GitHub Actions CI（本轮）** | push + PR | 跨设备 check/test/clippy/fmt + smoke |
| `make build-onnx`（Day 3 已有） | 开发者手动 | onnx feature 编译 |

CI 是"防回归"的最后一道；本地 pre-commit 是"防错入"的第一道。互不替代。
