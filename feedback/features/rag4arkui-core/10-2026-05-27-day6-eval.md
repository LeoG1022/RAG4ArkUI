### 关键算法

**recall@k**：
```rust
hit_count = |returned_top_k ∩ ground_truth|
recall_at_k = hit_count / |ground_truth|  // 空 GT 时 0
```

**MRR@k (Mean Reciprocal Rank)**：
```rust
mrr_at_k = 1 / first_gt_rank   // 若 top_k 中无 GT 则 0
```

**latency p50/p99**：
- 单 query 测 `Instant::now()` 到完成时间
- 全集 sort + 取索引 floor(len*0.5) / floor(len*0.99)

### 集成位置（不破坏现有 trait）

`Evaluator::with_reranker()` 让评估器自己管 retriever → reranker 链式调用：
- 不开 reranker：retriever.retrieve(k) → 直接计算
- 开 reranker：retriever.retrieve(pre_rerank_k=50) → reranker.rerank(k=5) → 计算

与 CLI cmd_query 的流程对齐，避免分歧。

### 控制台输出 + 报告双轨

控制台（stdout）：5 行 summary 让用户秒看
```
✅ 评估完成
   total queries  : 8
   avg recall@5   : 0.625
   avg MRR@5      : 0.583
   avg latency    : 12.3 ms
   p50 latency    : 11.0 ms
   p99 latency    : 18.0 ms
   report saved   : reports/eval-<timestamp>.md
```

Markdown 报告（reports/）：3 节
1. 整体指标表
2. 每 query 详情表
3. **失败 query 详情**（recall<1.0 的逐条展开 + 返回 top-k + 漏命中 GT）

### 评估集 schema（YAML）

```yaml
- id: q1
  query: "ArkUI-X 如何实现下拉刷新"
  relevant:
    - "list.md#List/下拉刷新@10"
  notes: "可选备注"
```

chunk_id 与 `MarkdownChunker::make_id` 严格对齐：`<source>#<heading_path>@<start_line>`。

### 8 条 sample query 设计（覆盖 4 类风格）

| 类别 | sample id | 用途 |
|---|---|---|
| 自然语言 | q-routing-push / q-list-refresh / q-state-mgmt | 用户口语场景 |
| 精确 API 名 | q-routing-api | 关键词匹配（BM25 应强） |
| 迁移场景 | q-kmp-migration | 方案 §2.6 优先场景 |
| 错误修复 | q-error-fix | 方案 §2.6 最易切入场景 |
| 负样本 | q-negative-1 | 完全无关，期望 recall=0（防过敏） |
| 多 GT 容忍 | q-multi-gt | OR 关系测试 |

### 报告文件命名

`reports/eval-<unix_ts>-<embedder>-<bm25>-<rerank>-<k>.md`

**API 选型**：
- `Evaluator::new(retriever).with_k(k).with_reranker(rr).with_config(c).run(&queries)` —— builder 链
- `EvalSummary` 全 derive Serialize（未来导出 JSON / Codecov 用）
- `render_markdown(&summary, run_at)` —— 报告与数据分离

**算法**：
- `partial_cmp` + `unwrap_or(Equal)` 防 NaN 排序 panic
- p99 index = `floor(len * 0.99) - 1`，小集合（n<100）也合理
- recall 空 GT 时返回 0（防除零）

**集成测**：
- tempdir + 真实 MockEmbedder + 真实 HybridRetriever + 真实 Indexer 完整闭环
- 断言 q1/q2 recall>0.5（MockEmbedder 对原文 cosine=1） + q_miss recall=0
- 渲染报告含"失败 query 详情"节
