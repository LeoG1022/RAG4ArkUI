## 本轮目标

用户问的「mdBook 是什么 / tree-sitter 是什么 / MVP 是什么」之类概念解释 · 之前散落在对话历史 · 下次新读者来一遍 agent 又得现场生成。建立 `docs/concepts/` 子目录作为问答知识库 · 与 GLOSSARY（速查）+ feedback（交互归档）+ STATUS（单轮快照）+ ADR（架构决策）形成完整文档矩阵。

## Plan

> 必备节（FAIL 级 M-FEATURE-PLAN）

三层定位：
| 层 | 文件 | 内容 |
|---|---|---|
| 速查 | `docs/GLOSSARY.md` | 一句话术语对照（1-2 行）|
| 深度 | `docs/concepts/<term>.md` | 4 节模板（一句话 / 业界用法 / 项目里怎么接 / 类比）|
| 交互归档 | `feedback/{meta,features}/*` | 用户 ↔ agent 决策对话（保持现状）|

加新概念流程：用户问 → agent 答 → 用户说「归档」→ agent 5 件事（写 concepts/`<term>`.md + 更新 concepts/README.md + GLOSSARY 加链 + mdbook include + SUMMARY 加子项）。

不引入顶层新目录（不叫 `knowledge/` / `qa/` / `kb/` / `wiki/`）· 放 `docs/concepts/` 子目录 · 跟 ADR / ROADMAP / STATUS 同级邻居。
