### 脚本设计要点

1. **隔离临时目录** `/tmp/rag-smoke-$$/`，不污染用户的 `corpus/` 与 `corpus/_index/`
2. **可断言的固定 fixture**：2 份带 frontmatter 的 markdown（router.md + list.md）
3. **可控的断言对象**：
   - `files=2`（grep -E "^   files       : 2$"）
   - `chunks>=4`（router 2 个 heading + list 2 个 heading = 4 chunks 最少）
   - `Top-1 source == list.md`（因为查询语句是 list.md 里某 chunk 的原文 → MockEmbedder cosine=1）
4. **失败诊断**：`--keep` 选项保留临时目录；默认完成清理
5. **可选 verbose**：`--verbose` 打印 cargo 完整输出，default 只打印 tail 20 行
6. **前置检查**：cargo 必须装好，否则报 127

### MockEmbedder 阶段的断言基础

关键洞察：MockEmbedder 对**完全相同文本**返回 cosine=1。所以 smoke 查询语句 = 文档某 chunk 的逐字原文，可强断言 Top-1 是该 chunk。Day 3 接 OnnxEmbedder 后这种"逐字命中"会被语义相似度替代，smoke 断言改成"Top-3 包含某 chunk" 即可。

### 与 cargo test 的边界

- `cargo test --workspace` 测：trait 契约 / 算法（RRF / cosine）/ 集成（indexer e2e in tempfile）
- `make smoke` 测：**CLI 二进制实际行为**（subprocess 退出码、stdout 格式、JSON 产物文件、跨进程 load）
- 互不替代

## 改动要点

> API 选型 / 算法 / 关键决策 / 与上轮的差异

- shell 选型：bash（不用 sh 因为用了 `[[ ]]`、`<<<`、process substitution）
- 退出码协议：0=通过 / 127=cargo 缺失 / 其他=断言失败
- trap EXIT 处理清理 + 保留诊断目录
- grep 断言用 `-qE` 正则锚定（行首 + 字段精确匹配）防止字符串误判
- Makefile target `smoke` 依赖 `install-rust`（先校验 cargo 存在）
