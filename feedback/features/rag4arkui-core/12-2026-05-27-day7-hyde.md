### 设计：QueryEnhancer trait + 两个实现

```rust
#[async_trait]
pub trait QueryEnhancer: Send + Sync {
    async fn enhance(&self, raw: &str) -> Result<EnhancedQuery>;
    fn name(&self) -> &str;
}
```

- **PassthroughEnhancer**（core）：透传，不改写。作为默认值/关闭 enhance 时的占位
- **MockHydeEnhancer**（retrieval）：确定性规则按 intent 生成 ArkTS 风格"假代码"

### 范围切分

| 做 | 不做 |
|---|---|
| QueryEnhancer trait 放 core | 真实远程 LLM 调用（Week 3 单独切片） |
| MockHydeEnhancer 规则法生成假代码 | 自动学习 query 风格 |
| 集成到 query / eval CLI 全链路 | HyDE-only retriever（仍走 HybridRetriever 现有流水线） |
| Evaluator 加 `with_enhancer` builder | 单独的 enhance subcommand |
| EvalConfig 加 hyde 字段 + 报告标识 | 配置矩阵自动跑（Day 7 续） |

### MockHyde 算法（规则法 · 5 条 intent）

1. **classify_intent(raw)** —— 检测关键词：
   - "router" / "路由" / "跳转" → `ApiLookup`
   - "error" / "错误" / "修复" → `ErrorFix`
   - "kmp" / "android" / "ios" / "迁移" → `Migration`
   - "一多" / "断点" / "adaptive" → `Adaptive`
   - "组件" / "列表" / "刷新" → `NewComponent`
   - 其他 → `Generic`

2. **extract_entities(raw)** —— 取驼峰 / 含点 / 含数字的 token

3. **generate_hyde_code(raw, intent)** —— 按 intent 套对应 ArkTS 模板：
   - `ApiLookup`: `import router from '@ohos.router'; @Component struct NavExample { build() { Button('跳转').onClick(() => router.pushUrl(...)) } }`
   - `NewComponent`: `Refresh / List / ForEach` 模板
   - `ErrorFix`: 编译错误 fix 模板
   - `Migration`: `ViewModel → async/await` 模板
   - `Adaptive`: `GridRow / GridCol 断点` 模板
   - `Generic`: 通用 `@Component` 模板

### CLI 集成

```bash
arkui-rag query --text "如何下拉刷新" --k 5 --hyde mock
arkui-rag eval --queries ... --hyde mock --k 5
```

`--hyde` 默认 `none`（向后兼容）。`mock` 启用 MockHydeEnhancer。
未来 `remote --hyde-endpoint ...` 留 Week 3+ 接入真实 LLM。

### 报告 / 输出标识

- query 输出行加 `hyde=<name>` 标识
- eval 报告头打印 `hyde=mock-hyde-arkts`
- 报告文件名加 hyde 段：`reports/eval-<ts>-<embedder>-<bm25>-<rerank>-<hyde>-<k>.md`
