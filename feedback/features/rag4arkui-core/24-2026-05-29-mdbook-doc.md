## 本轮目标

收尾 Week 6 4/4：把零散在 docs/ 的 24 个 md 文件 + 9 个 crate README + 6 个 mapping 文件
组织成**一个可搜索的 mdBook 文档站**：

- 用户从一个 URL 入口（GitHub Pages）就能找到所有文档
- mdBook 内置全文搜索（elasticlunr）+ 主题切换 + GitHub 编辑链接

同时准备 1.0 release notes 草稿 · 用户改版本号 + push tag `v1.0.0` 即触发完整 release。

## Plan

> 必备节（FAIL 级 M-FEATURE-PLAN）

### 文档单一信任源策略

诚实选择：**单一信任源在 `docs/`，mdbook/src 用 `{{#include ../../docs/...}}` 内联**。

| 方案 | 优 | 劣 | 选 |
|---|---|---|---|
| A. 把 docs/ 拷到 mdbook/src/ | site 自洽 · 可部署独立 | 双倍存储 · 容易漂 | ❌ |
| **B. mdbook/src 用 `{{#include}}` 引 docs/** | 单一信任源 · 维护 0 重复 | 必须本地 build · GitHub Action build 时 docs/ 也要 checkout | ✅ |
| C. mdbook 干脆指向 docs/ 当 src | 极简 | 失去 src/ + SUMMARY.md 自由组织能力 | ❌ |

方案 B 在 `mdbook build` 时由 `links` preprocessor（默认启用）展开。GitHub Action `actions/checkout@v4` 必带整个 repo · docs/ 自然在场。
