## 决策示例二：元变更必须留 feedback

**结论**：修改 `.claude/`、`scripts/`、`AGENTS.md`、`CLAUDE.md` 等元文件时，必须同步提交 `feedback/meta/[N]-*.md`。

**理由**：
- 规则文件的演化逻辑容易在多轮迭代后失去上下文
- 强制归档使未来 agent 和人类能理解"为什么有这条规则"，而不只是"有这条规则"
- `feedback/meta/` 是完整的审计轨迹，可从中重建工程演化历史

**机械化保证**：
```bash
bash scripts/classify-change.sh   # 识别元变更
bash scripts/preflight.sh          # skill 入口强制：检查是否关联 feedback
# pre-commit hook 自动拦截：元变更无 feedback → FAIL
```
